# 留言墙（Note Wall）— 产品需求文档（PRD）

## 1. 项目概述

### 1.1 项目背景
构建一个轻量级的在线留言墙网站，用户可以匿名或署名发布留言，所有留言以卡片墙的形式展示，支持实时加载与交互。

### 1.2 目标用户
- 普通访客：浏览留言、发布留言
- 管理员：管理（删除/置顶）留言

### 1.3 技术栈
| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Next.js（React） |
| 样式方案 | Tailwind CSS |
| 数据库 | Supabase（PostgreSQL） |
| 部署托管 | Vercel |

---

## 2. 功能需求

### 2.1 P0 — 核心功能（MVP）

#### 2.1.1 留言展示（留言墙）
- 以瀑布流/网格卡片形式展示所有留言
- 每张卡片包含：昵称、内容、发布时间、头像（随机生成）
- 按时间倒序排列（最新留言在最前）
- 支持分页或无限滚动加载

#### 2.1.2 发布留言
- 提供发布表单，包含以下字段：
  - **昵称**（选填，默认为"匿名用户"）
  - **留言内容**（必填，限制 1–500 字）
- 发布成功后留言实时出现在墙上
- 基础内容过滤（XSS 防护、空内容校验）

#### 2.1.3 点赞功能
- 每条留言支持点赞
- 使用本地存储（localStorage）记录已点赞状态，防止重复点赞
- 显示点赞总数

### 2.2 P1 — 增强功能

#### 2.2.1 留言搜索
- 支持按关键词搜索留言内容或昵称

#### 2.2.2 留言统计
- 展示总留言数、今日新增数

#### 2.2.3 暗色模式
- 支持亮色/暗色主题切换
- 记住用户偏好（localStorage）

### 2.3 P2 — 未来迭代

#### 2.3.1 管理后台
- 管理员登录（Supabase Auth）
- 删除不当留言
- 置顶留言

#### 2.3.2 留言回复
- 支持对某条留言进行回复

#### 2.3.3 图片上传
- 留言支持附带一张图片（Supabase Storage）

## 3. 数据库设计（Supabase）

### 3.1 表结构

#### `messages` 表
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | UUID | PRIMARY KEY, DEFAULT gen_random_uuid() | 留言唯一标识 |
| nickname | VARCHAR(50) | NOT NULL, DEFAULT '匿名用户' | 昵称 |
| content | TEXT | NOT NULL, CHECK(length <= 500) | 留言内容 |
| avatar_seed | VARCHAR(20) | NOT NULL | 头像随机种子（用于生成头像 URL） |
| likes | INTEGER | NOT NULL, DEFAULT 0 | 点赞数 |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |
| is_pinned | BOOLEAN | NOT NULL, DEFAULT false | 是否置顶（P2） |

### 3.2 Row Level Security（RLS）策略

| 操作 | 策略 |
|------|------|
| SELECT | 公开，所有人可读 |
| INSERT | 公开，任何人可插入（通过 anon key） |
| UPDATE | 仅限点赞字段递增（通过 RPC 函数） |
| DELETE | 仅管理员（P2 阶段实现） |

### 3.3 Supabase RPC 函数

#### `increment_likes(message_id UUID)`
- 原子性递增指定留言的 `likes` 字段
- 避免客户端直接 UPDATE 带来的安全问题

## 5. API 设计

### 5.1 前端 ↔ Supabase 直连（通过 Supabase JS Client）

| 操作 | 方法 | 说明 |
|------|------|------|
| 获取留言列表 | `supabase.from('messages').select('*')` | 按时间倒序，分页 |
| 发布留言 | `supabase.from('messages').insert({...})` | 插入新留言 |
| 点赞 | `supabase.rpc('increment_likes', { message_id })` | 调用 RPC 递增 |
| 搜索留言 | `supabase.from('messages').select('*').ilike('content', '%keyword%')` | 模糊搜索（P1） |

### 5.2 环境变量
```
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
```

## 6. 部署方案

### 6.1 Vercel 部署步骤
1. 将代码推送到 GitHub 仓库
2. 在 Vercel 中导入该项目
3. 配置环境变量（`NEXT_PUBLIC_SUPABASE_URL`、`NEXT_PUBLIC_SUPABASE_ANON_KEY`）
4. 自动构建部署，每次 push 到 main 分支自动触发重新部署

### 6.2 域名
- 默认提供 `xxx.vercel.app` 域名
- 可选绑定自定义域名

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏加载 < 2s，留言墙渲染流畅 |
| 安全 | XSS 防护、RLS 策略、输入校验、Rate Limit（防刷留言） |
| 兼容性 | 支持 Chrome / Firefox / Safari / Edge 最新两个版本 |
| 响应式 | 适配桌面端和移动端 |
| 可用性 | 无需注册即可浏览和留言 |
